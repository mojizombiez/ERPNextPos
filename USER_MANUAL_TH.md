## 1. เริ่มต้นใช้งาน

MWinPOS คือโปรแกรม Point-of-Sale บน Windows ที่เชื่อมต่อกับ **ERPNext** โดยตรง ข้อมูลทั้งหมดถูกเก็บไว้ในฐานข้อมูล SQLite บนเครื่อง และจะซิงค์ใบขายไปยัง ERPNext โดยอัตโนมัติ

### การเปิดใช้งานครั้งแรก

เมื่อเปิดโปรแกรมเป็นครั้งแรก (หรือหลังทำ Hard Reset) ระบบจะสร้างบัญชีพนักงาน **Admin** ให้อัตโนมัติ

- **PIN เริ่มต้น: `123456`**
- หลังเข้าสู่ระบบ ควรไปที่ **Admin → พนักงาน** เพื่อเพิ่มบัญชีพนักงานจริง และเปลี่ยน PIN ของ Admin ทันที

## 2. การเข้าสู่ระบบด้วย PIN

ทุกครั้งที่เปิดโปรแกรมหรือหลังหน้าจอล็อก จะแสดงหน้ากรอก PIN

| การกระทำ | คำอธิบาย |
|---|---|
| กดตัวเลข 0–9 | กรอกรหัส 6 หลัก |
| **CLEAR** | ล้างตัวเลขทั้งหมด |
| ← (ลบ) | ลบเลขตัวสุดท้าย |
| ส่งอัตโนมัติ | ระบบตรวจสอบทันทีเมื่อกรอบครบ 6 หลัก |

- หาก PIN ถูกต้อง ระบบจะนำไปหน้า Checkout
- หาก PIN ผิด จะแสดงข้อความ **"รหัสไม่ถูกต้อง"** และล้างตัวเลข
- การทำรายการบางอย่าง เช่น การลบสินค้าออกจากตะกร้า จะต้องกรอก **PIN ผู้จัดการ** (Manager)

## 3. หน้าขายสินค้า (Checkout)

หน้าหลักสำหรับขายสินค้า แบ่งเป็น 3 ส่วน:

### 3.1 แถบหมวดหมู่ (ซ้าย)
- แสดงกลุ่มสินค้า (Item Groups) จาก ERPNext
- มีหมวดพิเศษ **"สินค้าหมด"** ถ้ามีสินค้าที่ถูกปิดชั่วคราว
- คลิกหมวดเพื่อกรองสินค้าในกริด

### 3.2 กริดสินค้า (กลาง)
- แสดงสินค้าที่ **พร้อมขาย** ในหมวดที่เลือก
- แต่ละการ์ดแสดง: รูปสินค้า, รหัสสินค้า, ชื่อ, และราคา (฿)
- **คลิกสินค้า** เพื่อเพิ่มเข้าตะกร้า (เพิ่มทีละ 1 ต่อการคลิก)
- **ปุ่ม ✖ บนการ์ด**: ปิดสินค้าชั่วคราว (Sold Out) | ปุ่ม **+** บนการ์ดสีเทา: เปิดสินค้าอีกครั้ง

### 3.3 ตะกร้า / ออเดอร์ (ขวา)

| องค์ประกอบ | การกระทำ |
|---|---|
| **＋ / － ** | เพิ่ม/ลดจำนวน |
| **ไอคอนถังขยะ** | นำสินค้าออก (ต้องใช้ PIN ผู้จัดการ) |
| **พิมพ์ครัว** | ส่งรายการไปยังเครื่องพิมพ์ในครัว |
| **ล้าง** | ล้างตะกร้าทั้งหมด (ต้องยืนยัน) |
| **ราคารวม / ภาษี / ยอดรวม** | คำนวณอัตโนมัติ (VAT 7% รวมอยู่ในราคา) |
| **แถวยอดค้างชำระ** | คลิกเพื่อเพิ่มการชำระเงิน |
| **ชำระเงิน** | ยืนยันการขาย |

### 3.4 สินค้าแบบจัดชุด (Product Bundles)
MWinPOS รองรับระบบจัดชุดสินค้า (Product Bundle) ของ ERPNext อย่างเต็มรูปแบบ สินค้าที่เป็นชุดจะมีป้าย "Bundle" กำกับ เมื่อเพิ่มลงในตะกร้าและทำการขาย ระบบจะจัดการรายการและหักสต็อกส่วนประกอบย่อยอย่างถูกต้องเมื่อส่งข้อมูลใบขายขึ้นเซิร์ฟเวอร์ ERPNext

## 4. การรับชำระเงิน

คลิกที่ **แถวยอดค้างชำระ** เพื่อเปิดหน้าต่างชำระเงิน

### วิธีการชำระเงินที่รองรับ

| วิธี | ไอคอน |
|---|---|
| เงินสด | 💵 |
| บัตรเครดิต/เดบิต | 💳 |
| QR / พร้อมเพย์ | 📱 |
| อื่นๆ (จาก POS Profile) | กำหนดเอง |

### บัตรของขวัญและคูปองเงินสด (Gift Cards & Vouchers)
MWinPOS รองรับระบบบัตรของขวัญ (Gift Card) และคูปองเงินสดจาก ERPNext โดยตรง
1. เลือกวิธีชำระเงินเป็น **Gift Card** ในหน้าชำระเงิน
2. กรอกหรือสแกน **รหัสบัตรของขวัญ**
3. ระบบจะยืนยันความถูกต้องและยอดคงเหลือกับเซิร์ฟเวอร์ ERPNext อัตโนมัติ ก่อนนำยอดมาหักลบกับยอดค้างชำระในออเดอร์ของคุณ

### การชำระแบบผสม (Split Payment)
1. เลือกวิธีชำระและกรอกจำนวนเงิน → กด **Apply**
2. ทำซ้ำสำหรับวิธีชำระอื่น
3. รายการชำระจะแสดงเป็นแท็บเล็กเหนือแถวยอดคงเหลือ
4. โฮเวอร์แล้วกด ✖ เพื่อยกเลิกรายการชำระนั้น

**เงินทอน** จะแสดงสีเขียวเมื่อจ่ายเกินยอดรวม

## 5. การจัดการหลายออเดอร์พร้อมกัน

แถบแท็บแนวตั้งทางขวาของตะกร้า ช่วยจัดการหลายโต๊ะหรือหลายออเดอร์พร้อมกัน

| การกระทำ | วิธี |
|---|---|
| **สลับออเดอร์** | คลิกหมายเลขแท็บ |
| **ออเดอร์ใหม่** | คลิกปุ่ม **＋** ท้ายแถบ |
| **ยกเลิกออเดอร์** | คลิก ✖ บนแท็บ (ต้องยืนยัน) |

แต่ละแท็บมีตะกร้า, ข้อมูลการชำระ, และหมายเลขออเดอร์เป็นของตัวเอง

## 6. ประวัติออเดอร์

ไปที่ **ออเดอร์** จาก Sidebar

- ค้นหาและกรองออเดอร์ตาม **ช่วงวันที่**, สถานะ, และอื่นๆ
- ดูรายละเอียดออเดอร์ทั้งหมด รวมถึงรายการสินค้า, การชำระ, และยอดรวม
- **คืนเงิน**: เปิดออเดอร์และกดคืนเงิน (สร้าง Sales Invoice ติดลบใน ERPNext)
- **พิมพ์ใบเสร็จ**: พิมพ์ซ้ำสำหรับออเดอร์ที่ผ่านมา
- **ซิงค์ด่วน**: บังคับให้ออเดอร์นั้นซิงค์ขึ้น ERPNext ทันที

## 7. การจัดการลิ้นชักเก็บเงิน (Cash Drawer)

ไปที่ **Drawer** จาก Sidebar

### เปิดลิ้นชัก
1. กรอก **ยอดเงินเริ่มต้น** (ทอนย่อย)
2. กด **เปิดลิ้นชัก** — ระบบบันทึกยอดเปิด
3. สร้าง POS Opening Entry ใน ERPNext (ถ้ากำหนด POS Profile)

### ระหว่างวัน
- ยอดขายเงินสดถูกติดตามอัตโนมัติ
- กดเปิดลิ้นชักจริงได้ที่ **ตั้งค่า → เครื่องพิมพ์ → เปิดลิ้นชัก**

### ปิดลิ้นชัก
1. นับเงินจริงและกรอก **ยอดเงินสดจริง**
2. กด **ปิดลิ้นชัก**
3. ระบบคำนวณส่วนต่างเงินสดและสร้าง POS Closing Entry ใน ERPNext
4. แสดงสรุปยอดขายแยกตามวิธีชำระ

## 8. การจัดการสินค้าคงคลัง (Stock)

ไปที่ **Stock** จาก Sidebar

- ดูรายการสินค้าพร้อมรหัส, ชื่อ, ราคา, และหมวดหมู่
- **เพิ่มสินค้า** ด้วยตนเอง (โหมด Standalone) หรือซิงค์จาก ERPNext (โหมด Online)
- **แก้ไข** ชื่อ, ราคา, บาร์โค้ด, หรือรูปภาพ
- **ลบ** สินค้าออกจากฐานข้อมูล
- **เปิด/ปิดสินค้า**: ทำเครื่องหมายว่าจำหน่าย/หมด
- **อัปโหลดรูปภาพ**: แนบรูปสำหรับแสดงบนกริด
- **พิมพ์ป้ายราคา**: พิมพ์ label พร้อมชื่อและบาร์โค้ด

## 9. การจัดการลูกค้า

ไปที่ **ลูกค้า** จาก Sidebar

- ดูลูกค้าที่ซิงค์จาก ERPNext
- **เพิ่ม** ลูกค้าใหม่ (ชื่อ, โทรศัพท์, อีเมล, ที่อยู่)
- **แก้ไข** หรือ **ลบ** ลูกค้าที่มีอยู่
- สามารถเลือกลูกค้าให้กับออเดอร์ระหว่างขายได้

## 10. แคมเปญและโปรโมชั่น

ไปที่ **แคมเปญ** จาก Sidebar

- ดูและจัดการ **กฎโปรโมชั่น** (ส่วนลด, ชุดสินค้า, ฯลฯ)
- โปรโมชั่นถูกคำนวณและใช้อัตโนมัติเมื่อตรงเงื่อนไขในตะกร้า
- แต่ละโปรโมชั่นกำหนดจำนวนขั้นต่ำ, ยอดสูงสุด/ต่ำสุด, และเงื่อนไขหมวดสินค้าได้

## 12. แผงผู้ดูแลระบบ (Admin)

ไปที่ **Admin** จาก Sidebar — ต้องใช้ **PIN ผู้จัดการ**

### การจัดการพนักงาน

| การกระทำ | คำอธิบาย |
|---|---|
| **เพิ่มพนักงาน** | กรอกชื่อ, ชื่อเล่น, PIN 6 หลัก, และบทบาท (แคชเชียร์ / ผู้จัดการ) |
| **แก้ไขพนักงาน** | อัปเดตข้อมูลหรือ PIN |
| **ลบพนักงาน** | ลบบัญชีออก |
| **เปิด/ปิดใช้งาน** | ปิดบัญชีชั่วคราว (ไม่สามารถล็อกอินได้) |

> **คำแนะนำ:** สร้างบัญชี **ผู้จัดการ** อย่างน้อย 1 บัญชีก่อนเริ่มใช้งานจริง เพื่อให้สามารถอนุมัติรายการพิเศษได้

### การจัดการและบำรุงรักษาข้อมูล (Data Management)
ในแผงผู้ดูแลระบบมีเครื่องมือขั้นสูงสำหรับดูแลความสมบูรณ์ของฐานข้อมูลในเครื่อง โดยไม่ต้องติดตั้งโปรแกรมใหม่:
- **ล้างข้อมูลซ้ำซ้อน (Clean Duplicates):** สแกนและลบข้อมูลสินค้าหรือลูกค้าที่ทำการซิงค์จาก ERPNext มาซ้ำซ้อนกันเพื่อแก้ปัญหาข้อมูลบกพร่อง
- **ล้างออเดอร์ค้างซิงค์:** ลบออเดอร์ที่มีปัญหาข้อผิดพลาดรุนแรงและไม่สามารถส่งขึ้นเซิร์ฟเวอร์ ERPNext ได้
- **สถิติฐานข้อมูล:** แสดงขนาดของฐานข้อมูลที่ใช้ไป (MB) รวมถึงจำนวนรายการของ สินค้า ลูกค้า และออเดอร์ทั้งหมดที่มีในระบบปัจจุบัน

## 13. การตั้งค่า (Settings)

ไปที่ **ตั้งค่า** จาก Sidebar — ต้องใช้ **PIN ผู้จัดการ**

### ทั่วไป (General)

| การตั้งค่า | คำอธิบาย |
|---|---|
| **ภาษา** | เปลี่ยนระหว่าง 🇺🇸 อังกฤษ และ 🇹🇭 ไทย |
| **บริษัท ERPNext** | ชื่อบริษัทสำหรับใบขาย |
| **คลังสินค้า** | สาขา/คลังที่ใช้กับเครื่องนี้ |
| **POS Profile** | ลิงค์กับโปรไฟล์ ERPNext (เลือกอัตโนมัติจะดึงคลัง & บริษัท) |
| **โหมดการทำงาน** | **Online** (ซิงค์ ERPNext) หรือ **Standalone** (เก็บข้อมูลในเครื่อง) |
| **ข้ามการอัปเดตอัตโนมัติ** | ปิดการตรวจสอบอัปเดตเมื่อเปิดโปรแกรม |
| **URL อัปเดต** | URL ของไฟล์ JSON สำหรับการอัปเดต OTA |

### การเชื่อมต่อ (โหมด Online เท่านั้น)

| การตั้งค่า | คำอธิบาย |
|---|---|
| **URL ERPNext** | เช่น `https://mysite.frappe.cloud` |
| **ชื่อผู้ใช้ / รหัสผ่าน** | Token API ของ ERPNext |
| **ทดสอบการเชื่อมต่อ** | ตรวจสอบว่า URL เข้าถึงได้ |
| **เข้าสู่ระบบ** | ยืนยันตัวตนและบันทึก Token |
| **ช่วงเวลาซิงค์** | ความถี่ของการซิงค์อัตโนมัติ (วินาที) |
| **Batch Size** | จำนวนออเดอร์ต่อการซิงค์แต่ละรอบ |
| **ซิงค์ด้วยตนเอง** | ซิงค์ข้อมูลค้างทั้งหมดทันที |
| **Pre-cache** | ดาวน์โหลดข้อมูลสินค้าและลูกค้าสำหรับโหมดออฟไลน์ |

### เครื่องพิมพ์ (Printers)

| การตั้งค่า | คำอธิบาย |
|---|---|
| **เครื่องพิมพ์แคชเชียร์** | พิมพ์ใบเสร็จลูกค้า |
| **เครื่องพิมพ์ครัว** | พิมพ์บิลครัว |
| **ใช้เครื่องพิมพ์เดียวกัน** | ส่งทั้งใบเสร็จและบิลครัวไปยังเครื่องเดียว |
| **ค้นหาเครื่องพิมพ์** | ตรวจจับเครื่องพิมพ์ Windows อัตโนมัติ |
| **ตรวจสอบสถานะ** | ตรวจว่าเครื่องพิมพ์ออนไลน์ |
| **เปิดลิ้นชัก** | สั่งเปิดลิ้นชักผ่านเครื่องพิมพ์ใบเสร็จ |

### ความปลอดภัย (Security)

| การตั้งค่า | คำอธิบาย |
|---|---|
| **Master PIN** | PIN สำรองสากล (ค่าเริ่มต้น `123456`) — **เปลี่ยนทันที!** |
| **เวลาล็อกอัตโนมัติ** | ล็อกหน้าจอหลังไม่มีการใช้งาน N วินาที (ค่าเริ่มต้น 300 = 5 นาที) |

### รูปแบบ (Appearance)
- เลือกจาก **21 ธีม** (10 มืด, 11 밝า) เช่น Midnight, Makro, Crimson, Mint ฯลฯ
- ธีมแสดงตัวอย่างก่อนบันทึก

### เครื่องสแกน (Scanner)
- ทดสอบการสแกนในช่องทดสอบ
- ระบบแยกแยะ **เครื่องสแกน Hardware** จากการพิมพ์ด้วยมือ โดยใช้ความเร็วการกดแป้น
- แสดงประวัติการสแกน 10 รายการล่าสุด

### หน้าจอ (Display)
- **เปิดจอลูกค้า**: เปิดหน้าต่างแสดงผลบนจอที่สอง
- **เลือกจอ**: เลือกว่าจอไหนแสดงผลลูกค้า
- **โหมดเต็มจอ**: เปิดโปรแกรมแบบเต็มจออัตโนมัติ

### โซนอันตราย (Danger Zone)

| การกระทำ | คำเตือน |
|---|---|
| **ล้างข้อมูลทั้งหมด** | ลบสินค้า ออเดอร์ และลูกค้า แต่**เก็บการตั้งค่า** ไว้ — **ไม่สามารถกู้คืนได้** |
| **Hard Reset** | ลบฐานข้อมูลทั้งหมดรวมถึงการตั้งค่า — **ไม่สามารถกู้คืนได้** |

## 14. จอแสดงผลลูกค้า (Customer Display)

เมื่อเปิดใช้งาน ระบบจะเปิดหน้าต่างเพิ่มเติมบนจอที่สอง แสดง:
- รายการสินค้าในตะกร้าและจำนวน
- ยอดรวมปัจจุบัน
- โลโก้/แบรนด์ร้าน

เหมาะสำหรับจอที่หันไปทางลูกค้าที่เคาน์เตอร์

## 15. เครื่องสแกนบาร์โค้ด

รองรับ **เครื่องสแกน USB HID** (เสียบแล้วใช้ได้เลย):
- บนหน้า Checkout สแกนบาร์โค้ดสินค้าได้ทันที
- สินค้าถูกเพิ่มเข้าตะกร้าอัตโนมัติ
- หากไม่พบบาร์โค้ด ระบบจะถามว่าต้องการสร้างสินค้าใหม่ด้วยบาร์โค้ดนี้หรือไม่
- ระบบตรวจจับเครื่องสแกนโดยใช้ **ความเร็วการกดแป้น** (< 50ms = เครื่องสแกน)

---

## 16. การซิงค์ข้อมูลและโหมดออฟไลน์

### โหมด Online
- ซิงค์อัตโนมัติตามช่วงเวลาที่กำหนด
- ดึงสินค้า, ลูกค้า, และโปรโมชั่นจาก ERPNext
- ส่งออเดอร์ที่ขายแล้วไปยัง ERPNext เป็น Sales Invoice
- หากไม่มีอินเตอร์เน็ต ออเดอร์จะถูกเก็บในคิวและซิงค์ภายหลัง
- **ตัวบอกสถานะซิงค์** ที่ Header แสดงสถานะและจำนวนข้อผิดพลาดแบบ Real-time

### โหมด Standalone
- ไม่ต้องเชื่อมต่อ ERPNext
- จัดการสินค้า พนักงาน และลูกค้าในเครื่องได้เลย
- ออเดอร์เก็บในเครื่องอย่างเดียว ไม่มีการสร้าง Sales Invoice

## 17. ความปลอดภัยและการล็อกอัตโนมัติ

- หลังจากไม่มีการใช้งาน N วินาที (ค่าเริ่มต้น 5 นาที) หน้าจอจะล็อกอัตโนมัติ
- แสดงหน้ากรอก PIN เพื่อกลับเข้าสู่ระบบ
- การดำเนินการพิเศษ เช่น ลบสินค้าจากตะกร้า, เข้าหน้า Settings/Admin, ปิดลิ้นชัก — ต้องใช้ **PIN ผู้จัดการ**

## 18. การแก้ไขปัญหาเบื้องต้น

| ปัญหา | วิธีแก้ |
|---|---|
| **"รหัสไม่ถูกต้อง" ตอนใช้งานครั้งแรก** | ใช้ PIN `123456` (Admin เริ่มต้น) ถ้ายังไม่ได้ผล ตรวจสอบว่าโปรแกรมรีบิลด์หลัง update |
| **ไม่มีสินค้าแสดง** | ไปที่ ตั้งค่า → การเชื่อมต่อ → ซิงค์ด้วยตนเอง หรือตรวจสอบ API |
| **เครื่องพิมพ์ไม่พิมพ์** | ตั้งค่า → เครื่องพิมพ์ → ค้นหาเครื่องพิมพ์ แล้วตรวจสอบสถานะ |
| **ซิงค์ไม่ได้** | ตรวจสอบ URL และ Token ที่ ตั้งค่า → การเชื่อมต่อ → เข้าสู่ระบบ |
| **หน้าจอล็อกกะทันหัน** | กรอก PIN เพื่อปลดล็อก และปรับค่า Auto-Lock ที่ ตั้งค่า → ความปลอดภัย |
| **จอลูกค้าไม่เปิด** | ตั้งค่า → หน้าจอ → ค้นหาจอ เลือกจอที่ต้องการ แล้วกด Launch |
| **เข้าโปรแกรมไม่ได้เลย** | ทำ **Hard Reset** ที่ ตั้งค่า → โซนอันตราย จากนั้นใช้ PIN `123456` |
| **ออเดอร์ค้างไม่ซิงค์** | ไปที่ ออเดอร์ เปิดออเดอร์นั้น แล้วกด **ซิงค์ด่วน** |
